## 背景

用户要求把 `原图/`、`中间图/`、`产出图/` 下的内容全部改为 Git 忽略，并确认项目 skill 中涉及目录的地方是否需要同步更新。

## 关键过程

- 检查 `.gitignore`、`AGENTS.md` 和 `masked-imagegen-protection` skill。
- 确认三类图片目录中的实际文件仍被 Git 追踪，不能只靠新增忽略规则解决。
- 确认 skill 没有引用旧的中文目录，但示例命令仍使用旧的通用示例路径，需要同步改到当前项目结构。
- 准备为三类图片目录加入 `.gitkeep` 占位文件，仅保留目录结构进入版本库。

## 结果

- `.gitignore` 将忽略 `原图/`、`中间图/`、`产出图/` 下的实际内容。
- `AGENTS.md` 明确这三类目录默认只保留本地内容。
- `masked-imagegen-protection/SKILL.md` 的示例命令路径已对齐当前项目目录结构。

## 可复用经验

- 当工作目录需要本地保留但不进 Git 时，应同时做三件事：更新 `.gitignore`、移除已追踪文件索引、保留 `.gitkeep` 占位文件。
- skill 示例路径如果参与日常操作，应跟随项目目录重构同步更新，否则文档会误导后续执行。
