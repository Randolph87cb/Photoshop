# 整理项目目录并区分原图中间图产出图

- 日期：2026-05-05

- 相关 skill：`record-and-reflect-review`

## 背景

用户要求删除不必要内容，只保留工作需要的目录，并重新设计原图、产出图和中间图的目录结构，必要时同步更新 `.gitignore`。

## 关键过程

- 检查现有目录，确认原图散落在根目录，过程图和产出图混在旧目录中。
- 设计新的三层结构：`原图/`、`中间图/`、`产出图/`。
- 将根目录原图迁入 `原图/`，将 `assets/` 和 `生成式替换样例/` 迁入 `中间图/`，将 `朋友圈优化/` 迁入 `产出图/朋友圈版/`。
- 删除不再需要的旧目录入口和 Python 缓存目录。
- 同步更新 `AGENTS.md` 与 `.gitignore`。

## 结果

- 项目根目录不再散放图片文件。
- 原图、中间图、产出图分层明确，旧目录名已退出主结构。
- `.gitignore` 额外忽略了 `*.py[cod]` 和 `Thumbs.db`。

## 可复用经验

- 修图类项目应尽早按“源文件 / 过程文件 / 最终交付”拆分目录，避免阶段性文件和可交付文件长期混放。
- 做目录重构时，文档和忽略规则要一起更新，否则协作时容易重新回到旧目录习惯。
