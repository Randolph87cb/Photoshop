# 项目协作规则

## 目录结构

```text
P图/
├─ .codex/
│  └─ skills/
│     └─ masked-imagegen-protection/
│        ├─ SKILL.md
│        ├─ references/
│        └─ scripts/
├─ AI工作记录/
│  ├─ records/
│  │  └─ YYYY/MM/*.md
│  └─ skill-backlog.md
├─ 原图/
├─ 中间图/
│  ├─ 修图过程/
│  └─ 生成式替换样例/
│     └─ manual-mask/
├─ 产出图/
│  └─ 朋友圈版/
└─ .gitignore
```

## 目录说明

- `.codex/skills/masked-imagegen-protection/`：项目内沉淀的局部生成式修图 skill、参考模板和脚本。
- `AI工作记录/`：当前项目专用的 AI 工作记录与 skill 沉淀，不与其他项目混放。
- `原图/`：原始输入图片，只保留未经处理的源文件。
- `中间图/修图过程/`：修图过程中的阶段性结果、对齐结果和问题排查图。
- `中间图/生成式替换样例/`：生成式替换相关的参考图、裁切样例和过程预览，其中 `manual-mask/` 保存手工遮罩、对齐预览和区域变换参数。
- `产出图/朋友圈版/`：最终可交付或可直接发布的成图。

## 规则

- 修改项目前先查看这里的目录结构说明，避免把素材、产物、记录和 skill 文件放错位置。
- 新增、删除、重命名目录时，必须在同一次修改中同步更新本文件里的“目录结构”和“目录说明”。
- 目录变更如果影响到流程说明、示例路径、skill 引用或其他项目文档，也必须在同一次修改中同步更新对应文档，不能只改目录不改文档。
- `AI工作记录/` 只保存当前项目记录；新线程默认新增或更新对应摘要记录。
- 原图必须放在 `原图/`，中间产物必须放在 `中间图/`，最终交付图必须放在 `产出图/`，不要再散放到项目根目录。
- 提交前确认缓存文件、临时文件和无关产物没有被误加入版本库。
